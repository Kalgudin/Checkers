Или более простой подход: каждый ход - одно перемещение. Если после перемещения шашка оказалась на поле, где она может бить дальше, то игрок обязан продолжить, и ему предлагается сделать следующий ход той же шашкой. Ввод последовательно. Это удобнее для пользователя.

Реализуем так: функция make_move(start, end) проверяет, является ли ход допустимым. Если это рубка, то после перемещения проверяем, может ли эта шашка бить дальше. В случае, если игрок сделал рубку, но есть возможность дальше бить, и он ввел следующий ход, то продолжаем. Ход считается завершенным, когда либо нет дальнейших взятий, либо игрок сделал все.

Реализовать проверки:

Простая шашка: ход на одну клетку по диагонали вперед (для белых - уменьшение row, для черных - увеличение row). Если клетка пуста.

Взятие простой шашкой: прыжок через шашку противника на пустую клетку сразу за ней. Можно вперед и назад. Координаты: разница по row и col = 2. При этом шашка противника должна быть на средней клетке. После взятия удалить противника.

Дамка: ход на любое количество клеток по диагонали, пока не встретит препятствие. Взятие дамкой: прыжок через шашку противника на любую пустую клетку за ней (по диагонали). Между дамкой и целью не должно быть других шашек. После прыжка, шашка противника удаляется. Дамка может продолжать бить дальше с новой позиции, если есть еще шашки.

Обязательность взятия: если у игрока есть хотя бы одно взятие, он не может делать простой ход. Поэтому при выборе хода сначала проверяем, есть ли capturing moves. Если есть, то разрешаем только ходы, которые являются взятиями.

При множественных взятиях: если есть несколько вариантов взятий с разными шашками, игрок может выбрать любой. Если он выбрал шашку, которая может бить несколько раз, он обязан бить столько, сколько возможно (все доступные взятия этой шашкой). Но если после первого взятия у шашки есть несколько вариантов дальнейших взятий, игрок может выбрать любой. Надо реализовать проверку, что после каждого взятия, если есть продолжения, игрок должен продолжить бить этой же шашкой, иначе ход недействителен.

Проверка окончания игры: если у текущего игрока нет допустимых ходов (ни простых, ни взятий), то он проигрывает.

Интерфейс: цикл while not game_over: показываем доску, запрашиваем у текущего игрока ход. Можно поддерживать команды "quit" или "resign". Ввод: "a3 b4" или "a3 b4 c5" для цепочки. Разбираем строку на последовательность позиций. Если длина последовательности 2 - простой ход или одно взятие. Если больше - цепочка взятий. Проверяем, что каждое перемещение из одной позиции в следующую допустимо, и шашка на начальной позиции та же самая (для цепочки). Также нужно проверять, что после каждого перемещения, шашка могла бы сделать следующий ход (то есть есть возможность взятия). Если на каком-то этапе пользователь остановился, но есть еще возможные взятия, ход считается неполным и отклоняется.

Можно упростить: запрашивать ход по одному перемещению, и если есть продолжение, спрашивать "дальше?" и снова ввод. Это проще для пользователя и для программиста. Реализуем так: в основном цикле, если игрок выбирает взятие (рубку), то после применения хода проверяем, может ли та же шашка бить дальше. Если да, то продолжаем цикл для того же игрока, запрашивая следующий ход с той же шашки. Игрок может ввести следующий ход или, если нет возможности, то ход завершен. Но так как обязательность, если есть возможность, игрок обязан ее использовать, поэтому если он пытается завершить ход, не использовав все взятия, то это ошибка. def move_piece(self, from_pos, to_pos):
from_col = ord(from_pos[0]) - ord('a')
from_row = 8 - int(from_pos[1])
to_col = ord(to_pos[0]) - ord('a')
to_row = 8 - int(to_pos[1])

if self.board[from_row][from_col] == ' ':
return False

piece = self.board[from_row][from_col]
self.board[from_row][from_col] = ' '
self.board[to_row][to_col] = piece
return True

def get_piece_at(self, pos):
col = ord(pos[0]) - ord('a')
row = 8 - int(pos[1])
return self.board[row][col]

Проверим, есть ли у игрока ходы вообще. Если нет - победа противника.
Запросим ввод. Если "quit" - выход.
Разобьем на позиции. Проверим валидность последовательности.
Если валидно, применим, сменим игрока, иначе сообщим об ошибке и повторим.
Нужно также учесть, что после применения последовательности, может оказаться, что у противника нет ходов, тогда игра заканчивается.
